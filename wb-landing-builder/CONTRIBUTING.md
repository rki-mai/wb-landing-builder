# Запуск в docker контейнере

Для того, чтобы запустить сервис в docker-контейнере, необходимо
для начала собрать docker-образ. Для этого необходимо:

1. Перейти в директорию проекта.
3. Собрать образ через следующую команду

   ```shell
   docker build wb-landing-builder/ -t wb-landing-builder:dev
   ```

В результате сборки появится образ с тегом `wb-landing-builder:dev`.
Сам docker-контейнер можно запустить через следующую команду:

```shell
docker run --detach --rm -p "8080:8080" wb-landing-builder:dev
```

## Конфигурация контейнера

Собранный образ сервиса поддерживает параметры для конфигурации. Все параметры задаются
в виде переменных окружения. Ниже перечислены поддерживаемые параметры.

### Конфигурация сервиса

- `PORT` - порт, по которому сервис будет прослушивать входящие запросы.
  Дефолтное значение `8080`.

- `ENVIRONMENT` - окружение, в котором будет работать сервис.
  Дефолтное значение `production`.

- `LOG_LEVEL` - минимальный уровень логирования, который будет записывать
  сервис в stdout / stderr. Дефолтное значение `info`.

- `READ_TIMEOUT` - таймаут на чтение входящих запросов. Дефолтное значение `10s`.

- `WRITE_TIMEOUT` - таймаут на запись исходящих данных. Дефолтное значение `10s`.

- `RATE_LIMIT` - максимальное значение RPS, которое поддерживает сервис.
  Дефолтное значение `100`.

### Конфигурация подключения к Mongo

В качестве хранилища данных сервис использует Mongo. Для настройки подключения
к Mongo образ поддерживает следующие параметры:

- `MONGO_HOST` - название хоста с Mongo. Дефолтное значение `mongo`.

- `MONGO_PORT` - порт, по которому Mongo прослушивает запросы. Дефолтное значение `27017`.

- `MONGO_USER` - имя пользователя, от лица которого будет осуществлена авторизация.
  Дефолтное значение `admin`.

- `MONGO_PASSWORD` - пароль от пользователя, от лица которого осуществляется авторизация.
  Дефолтное значение `admin`.

- `MONGO_DATABASE` - название базы данных Mongo, которое будет использоваться со стороны сервиса.
  Дефолтное значение `storage`.

- `MONGO_MAX_CONNECTIONS` - максимальное количество одновременных соединений с Mongo, которое
  поддерживает сервис. Дефолтное значение `100`.

- `MONGO_TTL_DAYS` - ttl документов в днях. Дефолтное значение `30`.

## Запуск через docker compose

Чтобы не приходилось настраивать инфраструктуру для сервиса вручную, проект поддерживает
запуск сервиса вместе с настроенной инфраструктурой через docker compose. Для этого
необходимо:

1. Перейти в директорию проекта
2. Запустить docker compose через следующую команду:

   ```shell
   docker compose -f wb-landing-builder/docker-compose.yml up -d
   ```

> **NOTE:**
> Подымать docker compose окружение можно и без собранного образа. В этом случае
> при запуске образ автоматически соберется сам.
